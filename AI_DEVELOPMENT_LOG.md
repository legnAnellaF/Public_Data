# 🤖 AI 개발 비서 (Antigravity) 활동 기록 및 개발 일지

본 문서는 사용자와 AI(Antigravity)가 페어 프로그래밍을 통해 `Public Data Widget` 프로젝트를 구축하며 진행한 모든 주요 활동과 개발 내역을 하나의 파일로 요약한 것입니다.

## 📅 프로젝트 주요 마일스톤

### 1. 🏗️ 프로젝트 초기 세팅 및 아키텍처 설계
- **프론트엔드:** 검색 엔진(Google, Naver)의 우측 하단에 슬라이드업 형태로 나타나는 크롬 익스텐션(Chrome Extension) 기반 UI/UX 구축.
- **백엔드:** FastAPI를 이용한 비동기 REST API 서버 구축 및 CORS(교차 출처 리소스 공유) 설정 완료.

### 2. 🕷️ 지능형 공공데이터 크롤링 봇 구현
- **문제점:** 공공데이터포털(data.go.kr)의 기계적 접근 차단(DDoS 방어) 시스템으로 인해 단순 스크립트로는 엑셀 다운로드가 불가능한 문제 발생.
- **해결책:** **Playwright(플레이라이트)**를 도입하여 백그라운드에서 실제 크롬 브라우저를 구동하는 'Human-like' 온디맨드 크롤링 시스템 구현. 사용자가 [시각화] 버튼을 누를 때만 동작하여 차단 우회.

### 3. 🧠 1차 인공지능 시각화 엔진(V1) 개발
- 다운로드한 엑셀(CSV) 파일을 `pandas`로 읽어 들여, 의미 있는 '숫자 데이터(Y축)'와 '분류 기준(X축)'을 AI가 스스로 판별하는 자동 매핑 알고리즘 구현.
- `ImportError` 등의 로컬 모듈 참조 오류를 수정하여 안정성 확보.

### 4. 🚀 2차 지능형 시각화 엔진(V2) 대규모 업그레이드
- **다차원 콤보 차트(Combo Chart):** 엑셀에서 가장 중요한 수치 지표를 최대 3개까지 동시 추출. 하나는 막대(Bar), 다른 하나는 꺾은선(Line)으로 렌더링하고, 값의 스케일이 다를 경우 좌우 양쪽에 **듀얼 Y축**을 자동 생성하도록 개선.
- **페이지네이션(Pagination):** 데이터 항목이 100개가 넘어가도 UI가 망가지지 않도록, 한 번에 15개씩만 보여주는 [◀ 이전] / [다음 ▶] 슬라이드 넘기기 기능 구현.
- **백엔드 리팩토링:** 단일 데이터 강제 절삭(`head(15)`) 로직을 제거하고, 최대 100개의 데이터를 프론트엔드로 넘겨 동적 렌더링하도록 구조 변경.

### 5. 🎬 시네마틱 UX 및 사용자 경험(UX) 극대화
- **시네마틱 인트로:** 차트가 뜨기 전, 1위 지표(Max Value)를 거대하게 줌인(Zoom-In)하여 보여주는 극적인 인트로 연출 추가.
- **AI 랭킹 사유 추천:** 검색된 상위 5개의 엑셀 파일 아래에 "왜 이 데이터가 유용한지" 알려주는 AI 추천 가이드 문구(초록색 뱃지) 추가.
- **미니멀리즘 가독성:** 복잡한 X축 라벨을 가장 중요한 3개만 표시하고 나머지는 마우스 Hover(툴팁) 시에만 나타나도록 정리. 캔버스 높이를 400px로 확장하여 시인성 개선.
